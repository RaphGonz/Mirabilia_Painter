  <action>
    Create `renderer.py` with imports `import torch`, `import math`, `from config import IMG_SIZE` (D-08 flat import). Implement `draw(canvas, stroke_params) -> torch.Tensor` decorated with `@torch.no_grad()` (mandatory per D-04 — downstream Phase 2 uses the hard rasterizer as non-differentiable ground truth) using the RESEARCH.md Pattern 1 block.
    Steps: unpack `cx, cy, w, h, theta_01, r, g, b = stroke_params`; compute scalar `theta = theta_01.item() * math.pi` (D-03 half-turn mapping; `.item()` first per the ".item() before math.cos/sin" shared pattern — calling math.cos on a tensor raises TypeError). Read `H_px, W_px` from `canvas.shape[-2:]` and `device = canvas.device`. Build pixel grid with `torch.linspace(0.0, 1.0, H_px, device=device)` / for W, then `torch.meshgrid(ys, xs, indexing='ij')` (Pitfall 2 — always pass `indexing='ij'`). Translate (`dx = grid_x - cx`, `dy = grid_y - cy`), rotate into local frame with `cos_t`/`sin_t` from `math.cos`/`math.sin`, build the half-extent mask `(local_x.abs() <= w/2.0) & (local_y.abs() <= h/2.0)`, `.unsqueeze(0)` for broadcast, then `color = torch.stack([r, g, b]).view(3,1,1).expand_as(canvas)` and `return torch.where(mask, color, canvas)`.
    Do NOT use cv2 (D-02). Do NOT remove `@torch.no_grad()`. Do NOT omit `indexing='ij'`. Subpixel strokes returning the unmodified canvas is correct, not a bug — do not add special-casing to force them visible. Add `MIN_VISIBLE_STROKE_WIDTH = 2.0 / (IMG_SIZE - 1)` as a comment only (per RESEARCH open-question 2), not a config constant.
